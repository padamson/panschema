# Rust Codegen + Multi-Writer Fan-Out - Implementation Plan

**Feature:** Rust types writer and multi-writer dispatch for `panschema generate`

**User Story:** As a developer consuming a LinkML schema in a Rust application, I want `panschema generate` to emit a Rust module of types matching the schema ŌĆö structs for classes (with inheritance, mixins, and `slot_usage` honoured), enums for permissible values, doc-comments from LinkML descriptions ŌĆö so my application can build against versioned, pinned schema types alongside the existing HTML writer.

**Related ADR (if applicable):** None ŌĆö extends the writer architecture defined in [ADR-004: Reader/Writer Architecture](../adr/004-reader-writer-architecture.md).

**Approach:** Vertical Slicing with Outside-In TDD

---

## Why Now

Downstream consumers using `panschema add github:...` to pin a versioned LinkML schema currently get the schema's HTML docs but no language-native types. The first concrete consumer (t2t Ch 2 Phase 3) blocks on having Rust structs available so that an Axum application can hold parsed schema instances in typed memory.

This is also the first deliverable that exercises the multi-writer dispatch shape: `[generate.<name>]` blocks can now declare both `html` and `rust` keys, and `panschema generate` fans out across them. Once the fan-out lands, future writers (`shacl`, `sql`) drop in with the same plumbing.

## Design Decisions

### Output shape

- **Single flat module per schema.** `rust = "app/src/generated/scimantic.rs"` writes one Rust file. No submodules. Consumer chooses how to surface it (e.g. `pub mod scimantic;` or `include!(...)`).
- **`// @generated by panschema v<version>`** header line so hand-edits are clearly out-of-bounds.
- **Deterministic output.** Iterate over `BTreeMap` (already sorted) for classes/enums/slots. Generated file is byte-stable across runs ŌĆö idempotency falls out for free, no extra check needed.
- **String-templated codegen.** No `syn`/`quote`/`prettyplease` dep ŌĆö `format!` plus helper functions. Adds zero new dev-deps to panschema. Output is roughly idiomatic; downstream `rustfmt` pass is the consumer's option.

### LinkML construct ŌåÆ Rust mapping

| LinkML | Rust |
|---|---|
| Class (concrete) | `pub struct Foo { ... }` with `#[derive(Debug, Clone, Serialize, Deserialize)]` |
| Class (abstract) | Same struct; documented as abstract via doc-comment; no consumer enforcement in v0.1 |
| `is_a` parent | `pub trait Parent` with getter methods for that class's **direct** slots; child `impl Parent for Child`. Supertrait chain follows LinkML inheritance. |
| Mixin | Flatten the mixin's slots into the consuming class's struct. No trait emitted for mixins (Rust lacks multi-inheritance; trait-per-mixin compounded with is_a traits would be hard to follow). |
| Slot (required, single) | `pub field_name: T,` |
| Slot (optional, single) | `pub field_name: Option<T>,` |
| Slot (multivalued) | `pub field_name: Vec<T>,` |
| `slot_usage` override | The subclass's struct field uses the **refined** range/required/multivalued. Resolution is per-class, with usage applied over the inherited slot definition. |
| `any_of: [A, B, C]` | Per-slot `#[serde(untagged)] pub enum <Class><Slot>Value { A(Box<A>), B(Box<B>), C(Box<C>) }`. Box variants break recursive cycles. |
| LinkML enum | `pub enum EnumName { Variant, ... }` with `#[serde(rename = "<text>")]` per variant |
| `range: string` / `uri` / `uriorcurie` | `String` |
| `range: integer` | `i64` |
| `range: boolean` | `bool` |
| `range: float` / `double` | `f64` |
| `range: datetime` | `chrono::DateTime<Utc>` |
| `range: <classname>` / `<enumname>` | `T = ClassName` / `EnumName` |
| `description:` | `///` doc-comment, wrapped at ~80 chars |
| `identifier: true` slot | No special treatment in v0.1 ŌĆö a regular field |
| `class_uri: <prefix>:<term>` | Schema-local name wins for the Rust type name; URI prefix is LinkML metadata, not part of the Rust shape |

### Field-name mangling

LinkML uses lowerCamelCase (`wasGeneratedBy`); Rust idiom is snake_case. Slot names ŌåÆ snake_case for struct fields, with `#[serde(rename = "<linkML name>")]` so JSON/YAML round-trips against the LinkML wire format.

### Slice 6.2: RustWriter core ŌĆö structs, enums, slots, descriptions

**Status:** ŌÅ│ In Progress

**User Value:** Single-class and single-enum schemas produce idiomatic Rust types that compile against `serde` + `chrono`. No inheritance, no mixins, no `slot_usage`, no `any_of` yet.

**Acceptance Criteria:**
- [ ] `render_doc_comment` wraps LinkML descriptions into `///` lines
- [ ] `rust_type_for` maps LinkML primitives + class refs + enum refs to Rust types, honouring `multivalued` (Vec) and `required` (Option) combinatorics

### Slice 6.3: Inheritance traits, mixin flattening, `slot_usage`, `any_of`

**Status:** Ō¼£ Not Started

**User Value:** Real LinkML constructs round-trip. A schema with class hierarchies, mixins, refined slot ranges, and polymorphic `any_of` slot ranges produces Rust that captures the structure faithfully.

**Acceptance Criteria:**
- [ ] `is_a` chain ŌåÆ supertrait-linked traits with getter methods per direct slot; concrete struct flattens all inherited slots; `impl Trait for Child` provided
- [ ] Mixin flattening ŌĆö slots merged into the consuming class's struct; no trait emitted for the mixin itself
- [ ] `slot_usage` resolution: when a subclass refines a slot, the refined range/required/multivalued wins for that subclass's struct field. Walk `is_a` chain + mixins, collect slot definitions, apply `slot_usage` last-writer-wins per slot name.
- [ ] `any_of` per-slot enum unions with `#[serde(untagged)]` and `Box<T>` variants to break recursive cycles
- [ ] Focused unit test: `resolve_slots` for a hand-built `is_a` chain + `slot_usage` override returns the refined range, not the parent's
- [ ] Extended fixture exercising each construct; snapshot + build-the-output tests still green

**Notes:**
- Method-bearing (not marker) traits ŌĆö picked over the marker-trait alternative so consumers can write trait-bounded functions that read parent-class slots generically.

## Out of Scope (deferred past this feature)

- **Newtype wrappers for URIs** (`pub struct Uri(String)`). v0.1 uses `String`; revisit if/when meaningful.
- **rustfmt post-processing of generated output.** Consumers can pipe it themselves.
- **Trait method bodies derived from LinkML expressions or pattern constraints.** Traits are pure getters in v0.1.
- **`shacl` / `sql` writers.** Land under the same `[generate.<name>]` fan-out when downstream needs them ŌĆö small marginal work once 6.1 is in.
- **Generic / parameterized LinkML constructs.** None used in scimantic@0.1.0; defer until a real schema needs them.

## Implementation Log

### Slice 6.1 ŌĆö Multi-writer fan-out + RustWriter stub

**Completed:**
- New `panschema/src/rust_writer.rs` ŌĆö `RustWriter` implementing the `Writer` trait, with `render(schema) -> String` separated from `write(schema, path)` so future slices can snapshot the rendered output without a tempdir.
- `panschema/src/lib.rs` exposes the module publicly.
- `panschema/src/io.rs` registers `RustWriter` in `FormatRegistry::with_defaults()`.
- `panschema/src/manifest.rs` `GenerateConfig` grows a `rust: Option<PathBuf>` field. The pre-existing `errors_on_unknown_writer_in_generate` test (which used `rust` as the unknown-writer example) now uses `shacl`, preserving its meaning.
- `panschema/src/main.rs` `generate_from_manifest` dispatches on `gen_cfg.rust` after `gen_cfg.html`. The "no outputs generated" hint mentions both writer keys.
- Four unit tests in `rust_writer.rs` cover header content, schema-version line conditional, `format_id`, and `write`'s parent-dir creation.
- One manifest unit test asserts `rust` parses alone and alongside `html`.
- Two CLI integration tests assert `panschema generate` writes both files when both writer keys are present, and runs `rust` alone when only that key is present.

**Design decisions captured here for the record:**
- The plan considered chrono as a direct panschema dep so the build-the-output test in 6.2 could rely on it. Decided against: the scratch crate in 6.2 can declare chrono in its own generated `Cargo.toml`. Keep panschema's dep surface tight.
- Considered a `marker trait per is_a parent` shape but picked method-bearing traits with getters. Marker traits give type membership but no useful API; getter-bearing traits let downstream code write trait-bounded helpers.
- Considered single-file vs split tests; followed the project's existing convention (unit tests inline in `mod tests`, CLI integration tests in `tests/integration.rs`). Slice 6.4 will pull its fixture-heavy tests into a dedicated `tests/rust_writer.rs` when it earns the split.
